网站速度直接影响用户体验和搜索引擎排名。Google 的研究表明，加载时间从 1 秒增加到 3 秒，跳出率增加 32%。以下是 10 个实用的优化策略。

## 1. 优化图片加载

图片通常占据网页 60% 以上的体积。使用 WebP/AVIF 格式、srcset 响应式图片、loading="lazy" 原生懒加载。

## 2. 合理使用字体

Web 字体是性能杀手。使用 `font-display: swap` 避免 FOIT，预加载关键字体，子集化中文字体。

## 3. 减少关键渲染路径

内联关键 CSS，延迟加载非关键 CSS。首屏需要的样式越少，页面渲染越快。

## 4. 利用浏览器缓存

为静态资源设置合理的 Cache-Control 头。版本化的文件名配合长缓存时间是最佳实践。

## 5. 代码分割与按需加载

不要让用户加载他们看不到的代码。使用动态 import 实现路由级别的代码分割。

## 6. 预连接与预加载

对第三方域名使用 `preconnect`，对关键资源使用 `preload`。

## 7. 减少 JavaScript 体积

审视每一个依赖。Tree shaking、minification、compression 是基础操作。

## 8. 优化 Core Web Vitals

LCP（最大内容绘制）应在 2.5 秒内，FID（首次输入延迟）应在 100ms 内，CLS（累积布局偏移）应在 0.1 以下。

## 9. 使用 CDN

CDN 将你的静态资源分布到全球节点，用户从最近的服务器获取内容，延迟大幅降低。

## 10. 持续监控

使用 Lighthouse CI、Web Vitals 库或 Sentry 等工具持续监控，在性能退化时及时发现问题。
