从初级到高级，每个阶段有不同的重点。分享我的思考和教训。

## 第 1-2 年：打好地基

这个阶段最重要的是基础：数据结构、算法、网络协议、操作系统原理。具体框架会过时，基础不会。花时间深入理解 JavaScript 的原型链、HTTP 缓存、SQL 索引。写代码是唯一的学习方式。GitHub 的绿点不重要，代码质量重要。

## 第 3-4 年：从写代码到做设计

开始承担模块级别的设计。不仅仅完成任务，而是思考为什么要这样做。学会写设计文档，用文字描述技术方案，和团队讨论 trade-off。这是从执行者到思考者的转变。参与 Code Review，读别人的代码，学会给出有建设性的反馈。

## 第 5-7 年：影响力和领导力

技术深度已经不是瓶颈。学会沟通，学会管理上级预期，学会在技术债务和新功能之间做权衡。开始指导初级开发者，参与面试。你的价值不再只是你写的代码，而是你对团队的影响。选择做什么比怎么做更重要。

## 持续学习

每天学一点，一年后你会惊讶自己的变化。读技术博客、看会议视频、参与开源项目。不要追逐每一个新框架——理解它们要解决什么问题比学会它们的 API 更有价值。

## 最重要的

照顾好你的身体和心理健康。长时间的加班不会让你成为更好的开发者。运动、睡眠、社交和编程一样重要。技术生涯是一场马拉松，不是短跑。
