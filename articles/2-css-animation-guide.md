CSS 动画是现代 Web 体验不可或缺的一部分。但写好动画不仅仅是让东西动起来——它关乎性能、可访问性和用户体验的微妙平衡。

## 为什么选择 CSS 动画而非 JS 动画？

CSS 动画在浏览器合成器线程上运行，不占用主线程资源。这意味着即使在 JS 执行繁重任务时，CSS 动画依然能保持 60fps 的流畅度。对于 `transform` 和 `opacity` 属性，浏览器可以进行 GPU 加速，性能优势更加明显。

## 关键帧设计的黄金法则

好的动画应该遵循自然的缓动曲线。`cubic-bezier(0.4, 0, 0.2, 1)` 是最接近真实物理运动的缓动函数——物体先加速再减速，而非突兀地开始和停止。动画时长也大有讲究：微交互控制在 150-250ms，入场动画控制在 300-500ms，超过 1 秒的动画会让用户感到不耐烦。

## IntersectionObserver：滚动触发动画

传统的滚动监听使用 `scroll` 事件，但它在主线程触发，容易造成性能瓶颈。`IntersectionObserver` 是浏览器原生提供的异步 API，当元素进入视口时触发回调，性能远超传统方案。

## 尊重用户的动效偏好

使用 `prefers-reduced-motion` 媒体查询，当用户在系统层面开启了「减少动效」时，应该大幅减弱甚至关闭动画。

## 实战案例：霓虹发光脉冲

通过 `box-shadow` 动画配合缓动函数，实现了呼吸般的脉冲效果，既吸引注意又不至于刺眼。
